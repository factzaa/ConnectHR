# 🚀 คู่มือทำให้ใช้งานได้จริง — ทีละสเต็ป

ทำตามลำดับนี้ 8 สเต็ป ใช้เวลารวมประมาณ 20–30 นาที (ครั้งแรก)
แต่ละสเต็ปมี **✅ จุดตรวจสอบ** ให้ยืนยันก่อนไปต่อ

---

## สเต็ป 1 — สร้างโปรเจกต์ Supabase (ฟรี)

1. เปิด https://supabase.com แล้วกด **Start your project** → ล็อกอินด้วย GitHub หรืออีเมล
2. กด **New project**
3. กรอก:
   - **Name**: `recruitment` (หรือชื่ออะไรก็ได้)
   - **Database Password**: ตั้งรหัสผ่านฐานข้อมูล แล้ว **จดเก็บไว้** (ใช้กรณีฉุกเฉิน)
   - **Region**: เลือก **Southeast Asia (Singapore)** (ใกล้ไทยสุด เร็วสุด)
4. กด **Create new project** แล้วรอ ~2 นาทีให้สถานะขึ้นเขียว

**✅ จุดตรวจสอบ:** เห็นหน้า Dashboard ของโปรเจกต์ และสถานะโปรเจกต์เป็น *Active/Healthy*

---

## สเต็ป 2 — สร้างตาราง + สิทธิ์ + ที่เก็บไฟล์ (รัน SQL)

1. เมนูซ้าย เลือก **SQL Editor** → กด **New query**
2. เปิดไฟล์ `supabase/schema.sql` ในโฟลเดอร์โปรเจกต์นี้ → คัดลอก **ทั้งไฟล์**
3. วางลงในช่อง SQL → กด **Run** (หรือ Ctrl/Cmd + Enter)
4. รอจนขึ้น **Success. No rows returned**

**✅ จุดตรวจสอบ:**
- เมนูซ้าย **Table Editor** → เห็นตาราง `applicants`
- เมนูซ้าย **Storage** → เห็น bucket ชื่อ `applications`

> รันซ้ำได้ปลอดภัย ถ้าเคยรันเวอร์ชันเก่ามาก่อน สคริปต์มีคำสั่งเพิ่มคอลัมน์ใหม่ให้อัตโนมัติ (ไม่ลบข้อมูลเดิม)

---

## สเต็ป 3 — คัดลอกกุญแจ API มาใส่ config.js

1. เมนูซ้าย **Project Settings** (รูปเฟือง) → **API**
2. คัดลอก 2 ค่านี้:
   - **Project URL** — เช่น `https://abcdefgh.supabase.co`
   - **anon public** (ใต้ Project API keys) — ข้อความยาวขึ้นต้นด้วย `eyJ...`
3. เปิดไฟล์ `config.js` แก้ 2 บรรทัดแรกให้เป็นค่าจริง:
   ```js
   SUPABASE_URL: "https://abcdefgh.supabase.co",
   SUPABASE_ANON_KEY: "eyJhbGciOi...ค่ายาว ๆ...",
   ```
4. (ทางเลือก) แก้ `COMPANY_NAME`, `TAGLINE`, และ `POSITIONS` (ตำแหน่งที่เปิดรับ) ให้ตรงบริษัทคุณ
5. บันทึกไฟล์

**⚠️ ใช้เฉพาะกุญแจ `anon public` เท่านั้น** ห้ามเอา `service_role` มาใส่ (อันตราย)

**✅ จุดตรวจสอบ:** ในไฟล์ `config.js` ไม่มีคำว่า `YOUR-PROJECT` หรือ `YOUR-ANON-KEY` เหลืออยู่

---

## สเต็ป 4 — สร้างบัญชีผู้ดูแล (แอดมิน) สำหรับหน้าหลังบ้าน

1. เมนูซ้าย **Authentication** → **Users** → กด **Add user** → **Create new user**
2. ใส่ **อีเมล** และ **รหัสผ่าน** ของแอดมิน → ติ๊ก **Auto Confirm User** (สำคัญ ไม่ต้องยืนยันอีเมล)
3. กด **Create user**

**✅ จุดตรวจสอบ:** เห็นบัญชีแอดมินในรายการ Users สถานะยืนยันแล้ว

---

## สเต็ป 5 — ทดสอบบนเครื่องก่อน (Local Test)

เปิดโฟลเดอร์โปรเจกต์ในเทอร์มินัล/Command Prompt แล้วรัน:
```bash
# ถ้ามี Python
python -m http.server 8000
```
หรือถ้ามี Node:
```bash
npx serve
```
จากนั้นเปิดเบราว์เซอร์:
- หน้า QR: http://localhost:8000/index.html
- หน้าสมัคร: http://localhost:8000/apply.html
- หน้าหลังบ้าน: http://localhost:8000/admin.html

**ทดสอบวงจรเต็ม:**
1. เปิด `apply.html` → กรอกข้อมูลทดสอบ 1 คน → กดส่ง → ต้องขึ้น **"ส่งใบสมัครเรียบร้อย"**
2. เปิด `admin.html` → ล็อกอินด้วยแอดมินจากสเต็ป 4 → ต้องเห็นใบสมัครทดสอบในตาราง
3. คลิกแถว → ดูรายละเอียด → เปลี่ยนสถานะ → กดบันทึก → ตัวเลขสถิติด้านบนต้องอัปเดต

**✅ จุดตรวจสอบ:** สมัคร → เห็นในหลังบ้าน → เปลี่ยนสถานะได้ ครบวงจร

> เปิดไฟล์ตรง ๆ ด้วย `file://` การอัปโหลดไฟล์และกล้องอาจไม่ทำงาน — ต้องผ่านเซิร์ฟเวอร์ (localhost) หรือ https เท่านั้น

## สเต็ป 6 — นำขึ้นออนไลน์ (Deploy ให้ได้ URL จริง)

**วิธีที่ง่ายที่สุด — Netlify Drop (ไม่ต้องติดตั้งอะไร):**
1. เปิด https://app.netlify.com/drop
2. **ลากทั้งโฟลเดอร์** `recruitment-pwa` ไปวางในหน้าเว็บ
3. รอสักครู่ จะได้ URL เช่น `https://random-name-123.netlify.app`
4. (ทางเลือก) กด **Site settings → Change site name** เพื่อตั้งชื่อให้จำง่าย

**ทางเลือกอื่น:** Vercel (`vercel` ในโฟลเดอร์) หรือ GitHub Pages (push ขึ้น repo แล้วเปิด Pages)

**✅ จุดตรวจสอบ:** เปิด `https://ชื่อไซต์.netlify.app/apply.html` บนมือถือได้จริง และปุ่ม **📷 ถ่ายรูป/สแกน** เปิดกล้องได้ (เพราะเป็น https)

---

## สเต็ป 7 — จำกัดโดเมน (ความปลอดภัย แนะนำ)

เพื่อให้แน่ใจว่าเฉพาะเว็บของคุณเรียกใช้ Supabase ได้:
1. Supabase → **Authentication** → **URL Configuration**
2. ใส่ **Site URL** เป็น URL จาก Netlify (เช่น `https://ชื่อไซต์.netlify.app`)

> ระบบปลอดภัยด้วย Row Level Security อยู่แล้ว: คนทั่วไปส่งใบสมัครได้อย่างเดียว อ่าน/แก้/ลบไม่ได้ — เฉพาะแอดมินที่ล็อกอินเท่านั้นที่เห็นข้อมูลและไฟล์

**✅ จุดตรวจสอบ:** ล็อกอินหลังบ้านบน URL จริงได้ปกติ

---

## สเต็ป 8 — ทำ QR Code แจกผู้สมัคร

1. เปิด `https://ชื่อไซต์.netlify.app/index.html`
2. หน้านี้จะสร้าง **QR Code** ชี้ไปหน้าสมัครให้อัตโนมัติ
3. กด **ดาวน์โหลด QR** เพื่อเซฟรูป หรือ **พิมพ์โปสเตอร์** เพื่อปรินต์ติดหน้าร้าน
4. หรือกด **คัดลอก** ลิงก์ แล้วส่งทางไลน์/เฟซบุ๊ก/อีเมล

**✅ จุดตรวจสอบ:** สแกน QR ด้วยมือถือเครื่องอื่น → เปิดหน้าสมัครได้ → ส่งใบสมัครแล้วเห็นในหลังบ้าน

## 🔧 ปัญหาที่พบบ่อย

| อาการ | สาเหตุ / วิธีแก้ |
|---|---|
| ส่งใบสมัครแล้วขึ้น error | ยังไม่ได้ใส่ค่าใน `config.js` หรือยังไม่ได้รัน `schema.sql` |
| ล็อกอินหลังบ้านไม่ได้ | ยังไม่ได้สร้าง user หรือไม่ได้ติ๊ก Auto Confirm (สเต็ป 4) |
| อัปโหลดไฟล์ไม่ได้ | ยังไม่ได้รัน `schema.sql` (ส่วน Storage) หรือเปิดด้วย file:// |
| กล้องไม่เปิด | ต้องเปิดผ่าน https (Netlify) หรือ localhost เท่านั้น |
| หน้าสมัครว่างเปล่า/ตำแหน่งไม่ขึ้น | `config.js` โหลดไม่ได้ ตรวจว่าไฟล์อยู่โฟลเดอร์เดียวกัน |
